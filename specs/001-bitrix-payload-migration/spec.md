# Feature Specification: Миграция товаров из Bitrix в Payload CMS

**Feature Branch**: `001-bitrix-payload-migration`  
**Created**: 2026-03-06  
**Status**: Draft  
**Input**: User description: "У меня есть интернет магазин на Bitrix, https://vsedlyavanny.kz/, у меня есть доступ к этому сайту, разработай решение которое позволит мигрировать, товары, картинки, описание, мигрировать это все на payload с загрузкой в базу, а картинки загрузить в blob storage. задача понять что за версия битрикс подключиться через рест api настроить подключение подключиться и перенести все товары картинки категории все сюда на пайлот через api" (Вся спецификация должна быть на РУССКОМ языке)

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Первоначальное подключение к Bitrix (Priority: P1)

Как администратор системы, я хочу настроить подключение к REST API Bitrix, чтобы иметь доступ к данным магазина.

**Why this priority**: Это блокирующая задача. Без подключения миграция невозможна.

**Independent Test**: Запуск проверочного скрипта, который выводит версию Bitrix и общее количество товаров через API.

**Acceptance Scenarios**:

1. **Given** наличие URL магазина и токена/вебхука, **When** запуск скрипта подключения, **Then** система возвращает статус "Connected" и версию Bitrix.
2. **Given** неверные учетные данные, **When** попытка подключения, **Then** система выводит понятную ошибку авторизации.

---

### User Story 2 - Миграция категорий (Priority: P2)

Как администратор, я хочу перенести иерархию категорий из Bitrix в Payload, чтобы структура каталога сохранилась.

**Why this priority**: Товары привязываются к категориям, поэтому категории должны быть созданы первыми.

**Independent Test**: Сравнение количества и вложенности категорий в Bitrix и в коллекции Categories в Payload.

**Acceptance Scenarios**:

1. **Given** дерево категорий в Bitrix, **When** запуск миграции категорий, **Then** в Payload создаются записи с сохранением `parentId`.

---

### User Story 3 - Миграция товаров и изображений (Priority: P1)

Как администратор, я хочу перенести все товары (название, описание, цена, изображения) в Payload, чтобы запустить новый сайт.

**Why this priority**: Основная ценность фичи — наличие товаров на новой платформе.

**Independent Test**: Выборочная проверка 10 товаров в админке Payload на соответствие данным в Bitrix.

**Acceptance Scenarios**:

1. **Given** список товаров в Bitrix, **When** запуск миграции товаров, **Then** создаются записи в коллекции Products, изображения загружаются в Vercel Blob, а в базе сохраняются ссылки на них.

---

### Edge Cases

- **Дубликаты**: Что произойдет при повторном запуске миграции? (Ожидаемое поведение: обновление существующих по внешнему ID).
- **Битые ссылки**: Как система обрабатывает отсутствие файла изображения в Bitrix?
- **Большой объем данных**: Как система справляется с миграцией 10,000+ товаров (пагинация API).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Система ДОЛЖНА поддерживать аутентификацию в Bitrix REST API через Входящий вебхук (предоставляемый пользователем URL с секретным ключом).
- **FR-002**: Система ДОЛЖНА сопоставлять поля Bitrix (NAME, PREVIEW_TEXT, DETAIL_TEXT) с полями коллекции Products в Payload.
- **FR-003**: Система ДОЛЖНА загружать изображения товаров в Vercel Blob Storage и связывать их с записями Media в Payload.
- **FR-004**: Система ДОЛЖНА сохранять иерархию категорий.
- **FR-005**: Система ДОЛЖНА сохранять внешний ID (Bitrix ID) в Payload для предотвращения дублирования.

### Key Entities *(include if feature involves data)*

- **BitrixClient**: Модуль для работы с API Bitrix (версионность, пагинация).
- **MigrationManager**: Логика обхода данных и сопоставления полей.
- **MediaUploader**: Инструмент для скачивания из Bitrix и загрузки в Vercel Blob.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 100% товаров из Bitrix перенесены в Payload.
- **SC-002**: Все изображения доступны по ссылкам Vercel Blob после миграции.
- **SC-003**: Время миграции 100 товаров не превышает 5 минут.
- **SC-004**: Структура категорий полностью идентична исходной.
